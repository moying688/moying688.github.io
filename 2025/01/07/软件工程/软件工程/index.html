软件工程

软件危机

介绍

在软件的开发和维护的过程中产生的一系列的严重的问题

原因

  1. 软件是逻辑部件，不是物理部件
  2. 轻视维护
  3. 软件规模庞大
  4. 在对需求不明确的情况下，早早着手编码开发
  5. 没有认识到程序只是软件的组成部分之一
  6. 软件变化快

特点

  1. 文档不合适
  2. 质量不可靠
  3. 需求不明确
  4. 往往不可维护
  5. 成本逐年增加
  6. 用户不满意

消除软件危机的途径

  1. 吸收和借鉴人类长期以来积累的开发经验、技术
  2. 开发和使用更好的软件工具
  3. 应该使用得到推广和认可的技术和开发工具

软件工程

介绍

软件工程是指导计算机软件开发和维护的一门工程学科，采用工程的概念、原理、技术和方法来开发与维护软件

特性

  1. 关注大型程序的构造
  2. 中心课题是控制复杂度
  3. 软件经常变化
  4. 开发软件的效率非常重要
  5. 和谐合作是关键
  6. 必须有效地支持它的用户

基本原理

  1. 分阶段的生命周期计划严格管理
  2. 坚持进行阶段评审
  3. 实行严格的产品控制
  4. 采用现代程序设计技术
  5. 结果应能清楚的审查
  6. 开发小组成员少而精
  7. 承认不断改进软件工程实践的必要性

软件工程方法学

  1. 传统方法学
  2. 面向对象方法学

软件工程的生命周期

三个时期和八个阶段

三个阶段

软件定义、软件开发、软件维护阶段

八个阶段

  1. 问题定义、可行性研究、需求分析
  2. 总体设计（概要设计）、详细设计、编码与单元测试、综合测试
  3. 软件维护

软件过程

为了获得高质量软件所需要完成的一系列任务的框架

模型

瀑布模型（文档驱动）

  • 阶段间具有顺序性和依赖性
  • 推迟实现的观点
  • 质量保证的观点
优点
  • 可强迫开发人员采用规范的方法
  • 严格规定了每个阶段必须要提交的文档。
  • 要求每个阶段交出的所有产品都必须经过质量保证小组的仔细验证
缺点
  • 文档驱动
  • 灵活性较差
  • 难以一开始就提出准确的需求
  • 不可逆
  • 实际项目很少按照模型顺序进行
  • 用户必须有耐心

快速原型模型

  • 线性开发
  • 快速
优点
  • 人员分配灵活，刚开始不用投入大量人力
  • 可以先发布部分功能给客户
缺点
  • 难以一开始就设计好准确的原型

增量模型

优点
缺点
  • 容易退化为边做边改模型

螺旋模型

优点
  • 适用于内部开发的大规模软件项目
  • 减少了过多测试或测试不足带来的风险
  • 有助于吧软件质量作为软件开发的一个重要目标
  • 风险驱动
缺点
  • 风险驱动
  • 软件开发人员需要具有丰富的风险评估经验
  • 过多的迭代次数会增加开发成本，延迟提交时间

喷泉模型

体现了面向对象软件开发过程迭代和无缝的特性
